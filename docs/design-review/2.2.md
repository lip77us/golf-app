## Bugs

- **B-01 · Hub team color wrong.** `Nassua Hub Screen.png`: Jim / Dave / Gil / Paul
  all read `Blue`. Per `Nassau Setup Screen 2.png` and `Score Entry Screen.png`,
  Gil & Paul are **Orange / T2**. The per-player team label on the hub roster
  isn't resolving the team color (likely defaulting to team1/blue).
- **B-02 · Landscape safe-area.** `Scorecard Horizontal.png`: the device
  home-indicator overlaps the `NET` / `STB` columns. Wrap the landscape
  scorecard in `SafeArea` (or pad the trailing edge) so the last columns clear
  the indicator.
- **B-03 · Score-entry chips clip.** `Score Entry Screen.png`: the gross-score
  chip row (1,2,3,4,5,6,7,…) runs past the right edge. Make it horizontally
  scrollable or a 3×3 grid. (= May audit B-06, still open.)
- **B-04 · (copy, FIXED)** Side-games subtitle in `casual_round_screen.dart`.
  Now: "Each settles separately and does not change how your main game is
  scored." — true for both overlay accumulators and capture add-ons (Spots).

## Drift (consistency)

- **D-01 · Title separators.** Mixed middle-dot vs em-dash: `Casual Round ·
  Course & Game` and `Scorecard — Group 1 · Strokes Off` (dot) vs `Nassau —
  Setup` (em-dash). Standardize on one (recommend ` · `).
- **D-02 · Title content + alignment.** Titles variously show the screen
  ("Casual Rounds", "Leaderboard"), the game ("Nassau", "Nassau (SO)"), or the
  course ("Tilden Park Gc"); some centered, some left-with-back-arrow. Pick a
  rule (e.g. screen name centered; context as a subtitle).
- **D-03 · Selected-state color.** Three treatments on adjacent controls:
  primary-game chip = dark-green fill / white text; group-size segment = light-
  green fill / dark text; handicap "SO Low" = light-green + check. One selected
  style.
- **D-04 · App-bar overload.** Score entry carries ~6 unlabeled icons; the
  leaderboard "⋯" menu duplicates its own toolbar icons. The existence of the
  "What do these buttons do?" sheet is the signal — consolidate (e.g. keep chat
  + leaderboard inline, move invite/share/help into "⋯" only).
- **D-05 · Button styles.** Dark-green filled, light-green FAB, grey-disabled,
  and outlined-green-on-green coexist with no stated rule. The hub group card's
  outlined-green-on-green-tint buttons are low-contrast. Define: filled = primary
  action, outlined = secondary, and a single disabled treatment.
- **D-06 · Input fields.** Login phone field uses a green floating-label outline;
  the name field is placeholder-only. Standardize on floating labels. (= May
  audit D-03.)
- **D-07 · "Gc" casing.** Course names render "Tilden Park Gc" / "Corica Park
  Gc". Data-side casing, but it reads as a typo — normalize "GC" on import/display.

- Recents quick-pick on course selection; real Halved logo in the drawer.
- Help affordances: the score-row "?" guide and the "What do these buttons do?"
  sheet are genuinely helpful.
- Clean surface tint + card system; release build (no DEBUG ribbon).
