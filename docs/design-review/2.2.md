## Bugs

- **B-01 · Nassau T1/T2 inverted in score entry. ✅ FIXED.** Score entry was the
  lone screen that inverted the numbering — `_teamLabelFor` badged team1 as "T2"
  and team2 as "T1", the per-hole grid + hole-outcome banner did the same, and
  the `_TeamBanner` matchup header listed team2 (orange) first. Result: "T2" read
  before "T1", contradicting the setup screen (`T1: Jim & Dave`) and the
  leaderboard (both use T1 = team1). Un-inverted all four spots; colors were
  always correct (team1 = blue, team2 = orange, from `nameColor`), so only the
  numbers + left/right order moved.
  Note: the `Blue` next to each player on `Nassua Hub Screen.png` is the **tee-box
  name** (all four on the Blue tees), not a team label — not a bug.
- **B-02 · Landscape safe-area. ✅ FIXED.** `Scorecard Horizontal.png`: the device
  home-indicator overlapped the `NET` / `STB` columns. Wrapped the landscape
  scaffold body in `SafeArea(top: false)`, which insets the horizontal scroll
  viewport off the notch + indicator without disturbing the scroll-to-hole math
  (it reads the live viewport dimension). Portrait already had `SafeArea`.
- **B-03 · Score-entry chips looked clipped. ✅ FIXED.** `Score Entry Screen.png`:
  the gross-score chip row appears cut off at the right edge. On inspection it was
  *not* a hard clip — `_InlinePicker` is a horizontal `ListView` that auto-scrolls
  to center on par (`_offsetFor`: startIdx = netPar − 3), so a chip is mid-cut at
  the edge by design. The real gap was discoverability. Added a scroll-driven
  edge-fade (`ShaderMask`, `BlendMode.dstIn`) that fades the leading/trailing chips
  only when there's more to scroll to — so it reads as scrollable for outlier
  scores. (= May audit B-06, now addressed.)
- **B-04 · (copy, FIXED)** Side-games subtitle in `casual_round_screen.dart`.
  Now: "Each settles separately and does not change how your main game is
  scored." — true for both overlay accumulators and capture add-ons (Spots).

## Drift (consistency)

- **D-01 · Title separators. ✅ FIXED.** The em-dash was already dominant (`X —
  Setup` across ~7 setup screens); the lone outlier was `Casual Round · Course &
  Game` / `· Players`, now `—`. Convention adopted: ` — ` separates the screen
  name from its primary qualifier; ` · ` stays for *secondary* qualifiers in
  subtitles (e.g. `Scorecard — Group 1 · Strokes Off`, `Observing · <group>`).
- **D-02 · Title content + alignment. ⚠️ MOSTLY NOT DRIFT (reassessed).** On
  inspection: alignment is iOS platform-default (centered) across the board, and
  the "tint" difference on Nassau Setup is M3's scroll-`surfaceTintColor` (varies
  by scroll position in the screenshots, not a per-screen choice). "What the title
  shows" (screen vs game vs course) is legitimately contextual. The only real
  nugget: **18 screens use the shared `GolfAppBar` vs 39 on raw `AppBar`** —
  `GolfAppBar` scale-downs long titles instead of ellipsizing. Standardizing on it
  is a deliberate 39-screen sweep (enhancement), not a quick cosmetic fix —
  deferred.
- **D-03 · Selected-state color. ✅ FIXED.** Three treatments on adjacent
  controls: primary-game chip = dark-green fill / white; group-size segment =
  light-green; handicap "SO Low" = light-green + check. Unified via a
  `SegmentedButtonThemeData` in `_halvedTheme` (selected = primary fill /
  onPrimary text) so every segmented control matches the canonical
  `GameSelectableChip`. Rule: **filled brand-green = selected**.
- **D-04 · App-bar overload.** Score entry carries ~6 unlabeled icons; the
  leaderboard "⋯" menu duplicates its own toolbar icons. The existence of the
  "What do these buttons do?" sheet is the signal — consolidate (e.g. keep chat
  + leaderboard inline, move invite/share/help into "⋯" only).
- **D-05 · Button styles. ✅ FIXED.** Dark-green filled, light-green FAB, and
  outlined-green-on-green coexisted with no rule. Adopted **filled brand-green =
  primary, outlined = secondary**: themed the FAB to primary fill (was M3's light
  default, so "New Casual Round" matches every other primary), and promoted the
  hub group-card "Enter Scores" from outlined to `FilledButton` (fixes the
  low-contrast outlined-on-tint and sets clear hierarchy over the "Edit …"
  buttons). Disabled state is M3's single default treatment.
- **D-06 · Input fields. ✅ NOT DRIFT (false positive).** Both the login phone
  field and the name field use the same shared `GolfTextField` with floating
  labels (`label:`). The screenshot just caught the phone field focused (label
  floated, green) and the name field empty (label resting, grey) — standard
  Material floating-label behavior, not an inconsistency. No change.
- **D-07 · "Gc" casing. ✅ FIXED.** Course names rendered "Tilden Park Gc" /
  "Corica Park Gc". Added a display-only `prettyCourseName()` (standalone
  `Gc`/`Cc` → `GC`/`CC`) applied at every client course-name parse (CourseInfo,
  CatalogCourse, CourseHit, the `*_course_name` summaries, the round-summary
  course strings, and the admin full-DB search). Safe — courses match server-side
  by `golf_api_id`, never by name.
